# Image Generation Guidelines

When generating images for content, follow these principles.

## Tufte-Style Design Principles

- **Clean background** - Off-white or neutral, not distracting
- **High data-ink ratio** - Every visual element should convey information
- **Minimal text** - Let the visual do the work
- **Clear labels** - When text is needed, make it readable and precise
- **No decoration** - No gradients, shadows, or flourishes that don't add meaning

## Good Prompts for Technical Content

- "A flowchart showing [process]"
- "A diagram illustrating [concept]"
- "An infographic about [topic]"
- "A simple icon representing [idea]"
- "A comparison chart of [A vs B]"
- "Architecture diagram for [system]"

## Style Modifiers

For consistency, consider adding:
- "Minimalist line drawing"
- "Flat design illustration"
- "Technical diagram"
- "Clean, simple, professional"
- "White/off-white background"
- "Black or dark gray lines and text"

## When to Generate vs. Use Screenshots

**Generate an image when:**
- You need a conceptual diagram
- You want to illustrate a process or flow
- You need an icon or simple visual

**Use screenshots when:**
- Showing actual UI or output
- Demonstrating real results
- Referencing specific code or data

## What NOT to Do

- No stock photo style images
- No overly complex or busy visuals
- No decorative elements that don't add meaning
- No photorealistic images unless specifically needed
- No text-heavy images (text should be in the article)
