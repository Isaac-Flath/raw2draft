Use this skill before drafting or editing blog posts, newsletters, social copy, scripts, or captions.

Workflow:

1. Read the active file from `~/.raw2draft/active-file` when the user refers to the current file.
2. If `agentkb` is available, run `agentkb search -s wiki "writing style"` and use the most relevant guidance.
3. If local wiki files exist, read `wiki/writing-with-zinsser.md`.
4. Edit for clarity, specificity, rhythm, and concision.

Style rules:

- Prefer direct, concrete sentences.
- Cut filler, throat-clearing, hedging, and generic AI phrasing.
- Preserve the writer's point of view and technical nuance.
- Use examples when a claim is abstract.
- Do not make the prose sound polished at the cost of sounding human.
